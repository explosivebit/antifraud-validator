## 1. Контекст и постановка задачи

**Запрос:** Преподаватель ведёт курс по агентской разработке. Для первой сессии нужен маленький готовый проект, на котором можно показать полный цикл: настройка CLAUDE.md → написание кода → commit → push → PR → review на GitHub. Проект должен быть понятен аудитории от нуля до senior/principal.

**Ограничение:** проект должен быть осмысленным (не Hello World), но помещаться в 30-40 минут live-демо.

---

## 2. Выбор задачи

### Что предложено

Три варианта:

| # | Вариант | Объём | Сложность |
|---|---------|-------|-----------|
| 1 | Валидатор транзакций (пороги) | ~50 строк, 3 файла | Низкая |
| 2 | CLI-антифрод фильтр (CSV) | ~100 строк, 4 файла | Средняя |
| 3 | Rule Engine (Strategy pattern) | ~150 строк, 6 файлов | Высокая |

### Метод выбора

Применены три метода рассуждения:

- **Абдукция:** урок проваливается, когда новички застревают на setup → Вариант 1 минимизирует риск
- **Дедукция:** цель урока — workflow, а не код → код должен быть тривиально читаемым → не Вариант 3
- **Индукция:** из практики: >3 файлов → новички теряются, <20 строк → senior скучают → оптимум 40-60 строк

### Решение

**Вариант 1 на JavaScript** (не Python) — ещё проще: нет venv/pip, `node` есть у всех, JSON нативный.

## 7. Написание кода

### validator.js (~100 строк)

- Два правила: `checkHighAmount` (>1M → BLOCKED), `checkMediumAmount` (>100k → SUSPICIOUS)
- Движок `validateTransaction`: BLOCKED-правила → SUSPICIOUS-правила → APPROVED
- CLI-режим: `node validator.js` читает `transactions.json` и выводит вердикты
- Паттерн `require.main === module` — файл и библиотека, и CLI
- Полный JSDoc для всех функций и типов

### transactions.json (5 транзакций)

| ID | Amount | Время | Ожидаемый вердикт (сейчас) | После ночного правила |
|----|--------|-------|----------------------------|----------------------|
| tx-001 | 5,000 | 14:30 | APPROVED | APPROVED |
| tx-002 | 250,000 | 11:00 | SUSPICIOUS | SUSPICIOUS |
| tx-003 | 2,500,000 | 09:15 | BLOCKED | BLOCKED |
| tx-004 | 75,000 | 03:45 | APPROVED | **SUSPICIOUS** ← diff |
| tx-005 | 100,001 | 22:10 | SUSPICIOUS | SUSPICIOUS |

### validator.test.js (18 тестов)

**Первая итерация:** 10 тестов — позитивные, негативные, граничные.

**Запрос на доработку:** Преподаватель потребовал полный аудит покрытия: edge cases, N+1/N-1, позитивные/негативные.

**Аудит выявил 7 пропущенных тестов:**
- N+1 и N-1 для обоих порогов (4 теста)
- Edge cases: amount=0, amount=1 (2 теста)
- Проверка rule name в результате engine (1 тест)

**Вторая итерация:** 18 тестов — все проходят.

## Ключевые решения

| # | Решение | Метод | Обоснование |
|---|---------|-------|-------------|
| 1 | Вариант 1 (валидатор) | Абд+Дед+Инд | Все три метода сошлись: минимум setup, максимум понятности |

| 6 | Public repo | Абдукция | Студенты клонируют без токена |
