## Вы здесь

Проект уже работает. Два правила проверяют подозрительные транзакции по сумме. Тесты зелёные, CI на GitHub запускается автоматически. Недостаёт **третьего правила** — ночных крупных переводов.

## Шаг 3 — спецификация

Функция `checkNightTransfer(transaction)` должна вернуть:

| Условие                                                                | Результат                                                 |
|------------------------------------------------------------------------|-----------------------------------------------------------|
| UTC-час в `[0, 6)` **И** `amount > 50_000`                             | `{ verdict: "SUSPICIOUS", rule: "checkNightTransfer" }` |
| Любое другое сочетание                                                 | `null`                                                    |

**Значения параметров:**
- UTC-час извлекается из `transaction.timestamp` (строка ISO 8601, например `"2026-04-15T03:30:00Z"`)
- `transaction.amount` — число
- Константы `SUSPICIOUS` и `APPROVED` уже экспортируются из `rules.js`

**Ограничения (из `CLAUDE.md`):**
- Plain functions, CommonJS (`require`), без классов
- Никаких новых npm-пакетов
- Строгое равенство (`===`)
- Без `try/catch` (нет известных ошибок для обработки)
- Без комментариев, повторяющих код

Если нарушает — скажите: «прочти CLAUDE.md внимательно и исправь нарушения».

## Шаг 6 — git, PR, мёрж

```bash

git commit -m "feat: implement checkNightTransfer rule body"
git push -u origin feat/night-transfer-rule-<your-github-username>
```

Затем:

1. Откройте ссылку, которую выдал `git push` (или зайдите в `Pull requests` → `New pull request`)
2. Базовая ветка: `main`, сравниваемая: ваша
3. Заголовок PR: `feat: implement night transfer rule (W1-L1)`
4. В теле PR — пара строк про то, что делали; приложите скриншот зелёных тестов
5. Дождитесь, пока `test` чек на GitHub станет зелёным (~10-15 сек)
6. **Squash and merge** → **Delete branch**

## Stretch-задачи (если справились за 20 минут)

1. **Тест на самую границу.** Добавьте тест на `00:00:00` UTC с `amount = 50_001` — это ранняя граница ночи и минимальная срабатывающая сумма. Убедитесь, что правило триггерит.
2. **Граница суммы.** Что произойдёт при `amount` ровно `50_000`? Правило **не должно** срабатывать (порог `>`, не `>=`). Напишите тест, который это проверяет.
3. **Документация.** Добавьте строку `checkNightTransfer` в таблицу «Pre-built Rules» в `docs/architecture.md`.
4. **Подумать:** если перевод сделан из Москвы (UTC+3) в 05:00 по местному, это 02:00 UTC. Правило сработает. Это правильное поведение для антифрода или нет? Запишите свой ответ в PR-описании.

| Симптом                                           | Что делать                                                              |
|---------------------------------------------------|-------------------------------------------------------------------------|
| `node: command not found`                         | Установите Node.js 20 LTS                                              |

| `27 total` но не `27 passed`                      | Прочитайте, какой тест красный; покажите Claude сообщение об ошибке     |

| CI красный на PR, локально зелёные                | Проверьте, что запушили актуальный коммит: `git log --oneline`          |
| «всё сломал, хочу сначала»                        | `git reset --hard origin/w1-l1` — вернёт вашу ветку к стартовой точке   |
